## Section 5: How we handle security

> 📝 Nội bộ: smart contract = code that holds money. Security không phải add-on, nó là core.

**Smart contracts hold money. Bugs don't get patched — they get exploited.**

Security is built into every line, not bolted on at the end:

**1. Defense in depth — multiple layers, not one audit.**
Unit tests → Integration tests → Fuzz testing → Static analysis → Internal audit → External audit recommendation. Every contract goes through all layers before mainnet.

**2. Battle-tested patterns — not invented here.**
We build on OpenZeppelin, Solmate, and our own Cyberk Smart Contract Library. Reinventing access control or reentrancy guards is how hacks happen. We use proven patterns.

**3. Gas optimization without sacrificing readability.**
A gas-optimized contract that nobody can audit is a liability. We optimize where it matters — hot paths, storage slots, batch operations — and keep the rest readable.

**4. Upgradeable when needed, immutable when possible.**
Not every contract needs a proxy. We choose the right pattern: transparent proxy (ERC-1967), UUPS, Diamond (ERC-2535), or immutable — based on the project's regulatory and business requirements.

**5. Security tooling — automated and continuous.**
- **Slither** — static analysis for common vulnerabilities
- **Mythril** — symbolic execution for deep bug detection
- **Echidna** — property-based fuzz testing
- **Foundry** — invariant testing and gas snapshots
- **OpenZeppelin Defender** — monitoring deployed contracts
